2. Хранение пользовательского справочника
В storage.ts добавь функции:

getTags() — читает пользовательский справочник из AsyncStorage (ключ user_tags). Если пусто — возвращает defaultPlaces и defaultActivities
saveTags(places, activities) — сохраняет справочник
addCustomTag(type: 'place' | 'activity', label: string, emoji: string) — добавляет пользовательский тег
deleteTag(type: 'place' | 'activity', id: string) — удаляет тег

Обнови структуру DayEntry добавив поля:
typescriptplaces: string[], // массив id выбранных мест
activities: string[], // массив id выбранных активностей
3. Экран выбора тегов при заполнении дня
В начале заполнения дня (до первого вопроса) добавь новый шаг — выбор тегов:

Заголовок "Где был и что делал?" с подписью "(необязательно)"
Два блока: "Места" и "Активности"
В каждом блоке горизонтальная прокрутка тегов
Каждый тег — кнопка с эмодзи и подписью, при нажатии выделяется цветом #3D9970
Можно выбрать несколько тегов из каждого блока
Кнопка "Далее" ведёт к первому вопросу

4. Редактор справочника в настройках
В SettingsScreen добавь новый пункт "Мои места и активности →". При нажатии открывает TagEditorScreen:

Два раздела: Места и Активности
Список всех тегов с кнопкой удаления (корзина) справа
Кнопка "Добавить место" и "Добавить активность" внизу каждого раздела
При добавлении — модальное окно с полем для названия и выбором эмодзи из списка популярных
Нельзя удалить все теги — минимум 3 в каждом разделе

5. В режиме просмотра дня
Если теги выбраны — показывай их горизонтальной строкой в самом верху записи после настроения. Если не выбраны — блок не показывается.
6. Обратная совместимость
Старые записи без полей places и activities читаются как пустые массивы.
