## 1. Концепция продукта

**День** — это мобильное приложение для ежевечерней микро-рефлексии. Идея простая: каждый вечер пользователь отмечает настроение, отвечает на несколько коротких вопросов о прошедшем дне, при желании добавляет теги места и активности, заметки, фото и поле «чем горжусь». Вся запись занимает 3–5 минут.

Приложение не требует регистрации, не отправляет данные на серверы и не показывает рекламу. Все записи и фотографии хранятся локально на устройстве.

**Целевая аудитория:** люди, которые хотят развить привычку ежедневной рефлексии без сложных инструментов — без папок, форматирования и обязательств.

**Ключевой принцип:** минимализм с глубиной по желанию. Базовая запись — одно настроение и пара вопросов; всё остальное (теги, интенсивность, фото, заметки, гордость) опционально.

## 2. Сценарии использования

### Сценарий A — Ежедневная запись (основной)

**Контекст:** вечер, пользователь открывает приложение впервые за день.

1. Экран «Сегодня» показывает дату и день недели.
2. Пользователь видит пять эмодзи-кнопок настроения и выбирает одно.
3. Приложение последовательно проводит через фазы заполнения:
   - **Настроение** — выбор одного из пяти уровней.
   - **Вопросы** — пять карточек с вопросами (что узнал, кого встретил, позитивный вопрос, негативный вопрос, вопрос дня).
   - **Заметки** — поле «Чем сегодня горжусь?», свободное описание дня и до 3 фотографий.
4. После завершения — анимированный экран подтверждения «День записан».
5. Опция «Посмотреть запись» открывает сохранённый день.

> На основном экране «Сегодня» заполнение идёт по упрощённому пути: настроение → вопросы → заметки. Расширенный путь с выбором тегов и отметкой интенсивности переживаний доступен на экране `/day-fill` (см. сценарий C — заполнение прошлого дня из календаря).

### Сценарий B — Просмотр и редактирование записи за сегодня

**Контекст:** пользователь уже заполнил дневник сегодня и возвращается в приложение.

1. Экран «Сегодня» автоматически переключается в режим просмотра.
2. Отображается карточка настроения с эмодзи и цветовым акцентом.
3. Чипы тегов места и активности (если выбраны).
4. Карточки с ответами на каждый вопрос (пустые ответы скрыты); под ответом — бейдж интенсивности, если был отмечен.
5. Карточка «Гордость дня», заметки «Итоги дня» и фото дня.
6. Любую карточку ответа, заметки и фото можно отредактировать прямо в режиме просмотра — изменения сохраняются сразу.
7. Кнопка «Поделиться днём» формирует картинку-карточку и открывает системное меню «Поделиться».
8. Стрелками можно перейти к предыдущему/следующему дню.

### Сценарий C — Просмотр прошлых дней через календарь

**Контекст:** пользователь хочет вспомнить, что происходило в конкретный день.

1. Открывает вкладку «Календарь».
2. Видит месячную сетку: дни с записями подсвечены цветными кружками (цвет = настроение).
3. Листает месяцы стрелками влево/вправо.
4. Нажимает на заполненный день — открывается детальная страница с полной записью.
5. Нажимает на прошлый день без записи — открывается расширенная форма заполнения (`/day-fill`). Будущие дни некликабельны.
6. Через поиск вверху экрана ищет день по тексту ответов или заметок — результаты показываются списком с подсветкой совпадения.

### Сценарий D — Воспоминания и серии

**Контекст:** пользователь ведёт дневник уже несколько месяцев.

1. Открывает вкладку «Воспоминания».
2. Видит карточку с текущей серией дней (streak) и личным рекордом.
3. Цитата дня.
4. Блоки «Неделю назад», «Месяц назад» и «Год назад» показывают соответствующие записи (если есть).
5. Кнопка «Удиви меня» открывает случайный день из всей истории записей.

### Сценарий E — Настройка уведомлений

**Контекст:** пользователь хочет получать напоминание в удобное время.

1. Открывает вкладку «Настройки».
2. В разделе «Уведомления» включает напоминания и задаёт время.
3. Появляется барабанный выбор времени (часы и минуты с шагом 5 минут).
4. Уведомление перепланируется автоматически.

### Сценарий F — Смена темы

**Контекст:** пользователь хочет тёмное оформление ночью.

1. В «Настройках» в разделе «Оформление» выбирает «Тёмная», «Светлая» или «Системная».
2. Тема применяется мгновенно ко всем экранам и навигации.

### Сценарий G — Персонализация тегов

**Контекст:** пользователь хочет настроить список мест и активностей под себя.

1. В «Настройках» открывает «Персонализация».
2. На экране редактора тегов добавляет свои теги (эмодзи из палитры + название), удаляет ненужные.
3. Меняет порядок тегов кнопками вверх/вниз.

### Сценарий H — Экспорт и восстановление данных

**Контекст:** пользователь хочет сделать резервную копию или перенести данные.

1. В «Настройках» в разделе «Данные» нажимает «Экспортировать данные» — формируется JSON-файл, открывается меню «Поделиться».
2. «Восстановить данные» — выбор JSON-файла резервной копии и импорт записей.

## 3. Функциональность

### 3.1 Запись дня

- **Выбор настроения** — 5 уровней с эмодзи: 😞 Плохо, 😐 Нейтрально, 🙂 Хорошо, 😄 Отлично, 🤩 Супер.
- **Два пути заполнения:**
  - **Основной (вкладка «Сегодня»)** — упрощённый: настроение → вопросы → заметки → завершение. Теги и интенсивность здесь не запрашиваются и сохраняются пустыми.
  - **Расширенный (`/day-fill`)** — настроение → теги → вопросы → заметки → завершение, с выбором тегов и отметкой интенсивности. Открывается при заполнении прошлого дня из календаря.
- **Вопросы дня** — «вопрос дня» назначается по порядковому номеру дня в году (циклически из общего пула вопросов). Дополнительно используются случайные «позитивный» и «негативный» вопросы.
- **Интенсивность переживаний** — для ряда ответов (узнал, встретил, позитив, негатив, гордость) можно отметить силу переживания по шкале 1–3 с эмодзи и подписью (в расширенном пути `/day-fill`).
- **Теги места и активности** — выбор из настраиваемых списков (по умолчанию 10 мест и 12 активностей; в расширенном пути `/day-fill`).
- **Поле «Чем горжусь»** — опциональное достижение дня с собственной интенсивностью.
- **Заметки дня** — свободное текстовое поле «Итоги дня».
- **Фото дня** — до 3 фотографий с камеры или из галереи; файлы сохраняются в файловой системе устройства.
- **Тактильный отклик** — лёгкая вибрация при выборе настроения, вибрация успеха при завершении (только iOS/Android).
- **Запись одна в день** — если запись уже сделана, форма заменяется режимом просмотра.

### 3.2 Просмотр и редактирование записей

- **Режим просмотра сегодня** — автоматически активируется, если запись уже есть; поддерживает навигацию к соседним дням.
- **Inline-редактирование** — ответы, заметки и фотографии можно менять прямо в режиме просмотра.
- **Бейджи интенсивности** — под ответами показывается отмеченный уровень переживания.
- **Детальная страница дня** — полная запись с карточками, цветом настроения и датой.
- **Скрытие пустых ответов** — если на вопрос нет ответа, карточка не показывается.
- **Поделиться днём** — генерация картинки-карточки (react-native-view-shot) и системное меню «Поделиться».
- **Цитата дня** — внизу записи показывается цитата, детерминированная по дате.

### 3.3 Календарь

- **Месячная сетка** — классическая календарная раскладка с началом недели в понедельник.
- **Цветовые точки настроения** — каждый заполненный день окрашен в цвет настроения.
- **Выделение сегодня** — текущий день обведён рамкой акцентного цвета.
- **Заполнение прошлого дня** — нажатие на прошлый день без записи открывает форму `/day-fill`. Будущие дни некликабельны.
- **Навигация по месяцам** — стрелки назад/вперёд, обработка перехода через год.
- **Поиск по записям** — строка поиска по тексту ответов и заметок с подсветкой совпадений и переходом к записи.
- **Легенда цветов** — объяснение шкалы настроений.

### 3.4 Воспоминания

- **Серия дней (streak)** — текущая непрерывная серия и лучшая серия за всё время.
- **Цитата дня** — детерминированная по дате.
- **Неделю / месяц / год назад** — соответствующие записи прошлого (если есть).
- **Случайный день** — кнопка «Удиви меня» выбирает случайную запись из всей истории.

### 3.5 Настройки

- **Уведомления** — включение напоминаний и настройка времени (барабанный выбор с шагом 5 минут).
- **Оформление** — три режима: Системная, Светлая, Тёмная.
- **Персонализация** — переход к редактору тегов мест и активностей.
- **О приложении** — переход на экран «Зачем дневник».
- **Данные** — экспорт всех записей в JSON и восстановление из файла резервной копии.
- **Диагностика** — проверка состояния локального хранилища.
- **Версия приложения** — «Den · v1.0.0» в подвале.

### 3.6 Уведомления

- Запрашиваются разрешения (iOS/Android).
- При включении напоминаний планируется ежедневное уведомление на заданное время.
- При смене времени — немедленная перепланировка.
- На web-версии уведомления не работают (платформенное ограничение).

### 3.7 Онбординг

- При первом запуске показывается экран онбординга, объясняющий идею приложения.
- Факт прохождения сохраняется локально, повторно не показывается.

## 4. Интерфейс

### 4.1 Навигация

Приложение построено на 4 вкладках (tab bar) в нижней части экрана:

| Вкладка | Содержание |
|---|---|
| Сегодня | Форма записи или просмотр дня |
| Календарь | Месячный вид с точками настроений |
| Воспоминания | Streak, цитата, недавние/прошлые записи, случайный день |
| Настройки | Уведомления, оформление, персонализация, данные, диагностика |

Отдельные роуты вне табов открываются поверх tab bar (презентация `card`):

- `/day-detail` — детальная страница конкретного дня.
- `/day-fill` — поэтапная форма заполнения дня.
- `/tag-editor` — редактор тегов мест и активностей.
- `/why-diary` — экран «Зачем дневник».

### 4.2 Фазы заполнения дня

Заполнение управляется переменной `phase`. Набор фаз отличается у двух экранов:

**Вкладка «Сегодня» (`(tabs)/index.tsx`)** — фазы `mood | questions | notes | done | view`:

- **`mood`** — выбор настроения (дата, ряд из 5 эмодзи, кнопка «Продолжить» активна после выбора).
- **`questions`** — пять карточек QuestionCard.
- **`notes`** — гордость, заметки и до 3 фото (NotesCard).
- **`done`** — анимация завершения «День записан» и кнопка «Посмотреть запись».
- **`view`** — режим просмотра/редактирования записи с навигацией по дням.

В этом упрощённом пути теги и интенсивность не запрашиваются (сохраняются пустыми/`null`).

**Экран `/day-fill` (`app/day-fill.tsx`)** — расширенный путь, фазы `mood | tags | questions | notes | done`:

- **`tags`** — выбор тегов места и активности.
- В фазе **`questions`** для части вопросов доступен выбор интенсивности (IntensityPicker).
- Остальные фазы аналогичны.

### 4.3 Карточка вопроса (QuestionCard)

- Счётчик и прогресс-точки
- Вопрос крупным шрифтом
- Многострочное текстовое поле
- Кнопки «Назад» (со второго вопроса) и «Далее» / «Готово»

Сама карточка не содержит выбора интенсивности — в расширенном пути `/day-fill` IntensityPicker выводится отдельным блоком рядом с вопросом.

### 4.4 Карточка дня (DayEntryView)

- Карточка настроения: эмодзи + название уровня, фон — полупрозрачный цвет настроения
- Чипы тегов места и активности
- Карточки ответов с inline-редактированием и бейджами интенсивности
- Карточка «Гордость дня»
- Заметки «Итоги дня»
- Фото дня (горизонтальная лента, просмотрщик, добавление/удаление)
- Кнопка «Поделиться днём»
- Цитата дня

### 4.5 Детальная страница (DayDetailScreen)

- Хэдер с кнопкой «назад» и датой
- Полная запись через DayEntryView

### 4.6 Карточка для шеринга (ShareCard)

- Компактное визуальное представление дня (настроение, ключевые ответы, дата), которое захватывается в PNG для функции «Поделиться».

## 5. Дизайн и оформление

### 5.1 Цветовая палитра

**Светлая тема:**

| Токен | Цвет | HEX |
|---|---|---|
| background | Светло-серый | `#F8F9FA` |
| foreground | Почти чёрный | `#1A1A1A` |
| card | Белый | `#FFFFFF` |
| primary (акцент) | Насыщенный зелёный | `#3D9970` |
| secondary | Светло-зелёный | `#EEF7F2` |
| muted | Серый | `#F0F0F0` |
| mutedForeground | Серый текст | `#888888` |
| border | Светлая граница | `#E8E8E8` |

**Тёмная тема:**

| Токен | Цвет | HEX |
|---|---|---|
| background | Почти чёрный | `#121212` |
| foreground | Светлый | `#F0F0F0` |
| card | Тёмно-серый | `#1E1E1E` |
| primary (акцент) | Мятный зелёный | `#5BAD8F` |
| secondary | Тёмный | `#2A2A2A` |
| border | Тёмная граница | `#2E2E2E` |

### 5.2 Цвета настроений

Одинаковы в обеих темах:

| Уровень | Эмодзи | Название | Цвет |
|---|---|---|---|
| 1 | 😞 | Плохо | `#7B8FA1` (синевато-серый) |
| 2 | 😐 | Нейтрально | `#A8B5C1` (светло-серый) |
| 3 | 🙂 | Хорошо | `#90C8A8` (мягкий зелёный) |
| 4 | 😄 | Отлично | `#5BAD8F` (зелёный) |
| 5 | 🤩 | Супер | `#3D9970` (насыщенный зелёный) |

### 5.3 Типографика

Шрифт: **Inter** (Google Fonts) — Regular 400, Medium 500, SemiBold 600, Bold 700.

### 5.4 Принципы дизайна

- **Скруглённые углы** — радиус 16–20px для карточек, 12–16px для кнопок.
- **Мягкие тени** — `shadowOpacity: 0.06–0.08` (iOS), `elevation: 2–4` (Android).
- **Акцент через прозрачность** — цвет настроения используется с альфа-каналом для фона карточек.
- **Анимации** — fade + scale при появлении и завершении.
- **Строгий ритм** — горизонтальные отступы 20px, вертикальные gap 12–20px.

| `react-native-view-shot` | Захват карточки дня в PNG |
| `expo-notifications` | Ежедневные напоминания |
| `expo-haptics` | Тактильная обратная связь |
| `@expo/vector-icons` (Ionicons) | Иконки |
| `@expo-google-fonts/inter` | Шрифт Inter |
| `@tanstack/react-query` | Провайдер запросов |
| `react-native-safe-area-context` | Безопасные зоны экрана |
| `react-native-gesture-handler` | Обработка жестов |
| `react-native-keyboard-controller` | Управление клавиатурой |

### Платформы

- **iOS** — через Expo Go или сборку
- **Android** — через Expo Go или сборку
- **Web** — ограниченно (без уведомлений и haptics; фото сохраняются как base64)

```
artifacts/den/
├── app/                          # Expo Router маршруты
│   ├── _layout.tsx               # Корневой layout с провайдерами и онбордингом
```

### 7.3 Поток данных

Глобального стора нет — каждый экран загружает данные самостоятельно через прямые вызовы `storage.ts` (обычно при фокусировке). Фотографии хранятся в файловой системе, а в записи дня сохраняются только пути/URI к ним.

```
AsyncStorage  ──┐
                ├─ storage.ts (saveDay / getDay / getAllDays / getTags / getStreak / ...)
файловая система ┘  + photoStorage.ts (savePhoto / deletePhoto)
        ↑ загрузка на экранах
             ↓ локальный useState
                  ↓ UI
```

### 7.4 Маршрутизация

Expo Router обрабатывает навигацию файловой системой:

- `/(tabs)/index` — главный экран (Сегодня)
- `/(tabs)/calendar` — Календарь
- `/(tabs)/memories` — Воспоминания
- `/(tabs)/settings` — Настройки
- `/day-fill` — форма заполнения дня
- `/day-detail?date=YYYY-MM-DD` — детальная страница
- `/tag-editor` — редактор тегов
- `/why-diary` — экран «Зачем дневник»

Ключевое правило: только файлы внутри `app/` являются маршрутами. Компоненты, контексты и утилиты вынесены в `src/`.

### 7.5 ThemeContext

```ts
type ThemeMode = "system" | "light" | "dark";
// Сохраняется в AsyncStorage, реактивно обновляет isDark во всём дереве.
```

Компоненты используют `useTheme()` и берут нужную палитру: `const theme = isDark ? colors.dark : colors.light`.

### 7.6 NotificationContext

1. Запрашивает разрешения на уведомления (iOS/Android).
2. Хранит флаг включения и время напоминания в AsyncStorage.
3. Планирует/перепланирует ежедневное уведомление при включении или смене времени.

## 8. Структура данных

### DayEntry — запись дня

```ts
type Intensity = 1 | 2 | 3 | null;

interface QuestionAnswer {
  question: string;
  answer: string;
  category: string;
}

interface DayAnswers {
  learned: string;             // Что узнал?
  met: string;                 // Кого встретил/вспомнил?
  positive: QuestionAnswer;    // Случайный позитивный вопрос + ответ
  negative: QuestionAnswer;    // Случайный негативный вопрос + ответ
  dayQuestion: string;         // Ответ на вопрос дня
}

interface DayEntry {
  date: string;                // "YYYY-MM-DD"
  mood: number;                // 1..5
  answers: DayAnswers;
  question: string;            // Вопрос дня на момент записи
  notes: string;               // Свободные «Итоги дня»
  photos: string[];            // До 3 путей/URI к фото
  proud: string;               // Чем горжусь
  learned_intensity: Intensity;
  met_intensity: Intensity;
  positive_intensity: Intensity;
  negative_intensity: Intensity;
  proud_intensity: Intensity;
  places: string[];            // ID выбранных тегов мест
  activities: string[];        // ID выбранных тегов активностей
}
```

**Обратная совместимость:** при чтении старых записей `parseEntry` нормализует данные — одиночное `photo` превращается в массив `photos`, устаревшие ключи `laughed`/`annoyed` мапятся в `positive`/`negative`, отсутствующие поля получают значения по умолчанию.

### Теги (UserTags)

```ts
interface TagItem { id: string; emoji: string; label: string; }
interface UserTags { places: TagItem[]; activities: TagItem[]; }
```

По умолчанию: 10 мест и 12 активностей. Палитра эмодзи для пользовательских тегов — 50 значений. Пользователь может добавлять, удалять и менять порядок тегов.

### Интенсивность (intensity.ts)

`INTENSITY_CONFIGS` описывает конфигурацию для ключей `learned | met | positive | negative | proud`. Каждая конфигурация задаёт варианты шкалы 1–3 (эмодзи + подпись), отображаемые через `IntensityPicker`.

### Ключи AsyncStorage

| Ключ | Содержание |
|---|---|
| `day_YYYY-MM-DD` (по дате) | Сериализованный `DayEntry` |
| Тема оформления | `"system"` / `"light"` / `"dark"` |
| Пользовательские теги | Сериализованный `UserTags` |
| Настройки уведомлений | Флаг включения, время, ID и дата планирования |
| Флаг онбординга | Пройден ли первый запуск |

### Вопрос дня — getDayQuestion

```ts
function getDayQuestion(date: Date): string {
  // Индекс по порядковому номеру дня в году, циклически по длине пула вопросов.
  return questions[(dayOfYear - 1 + questions.length) % questions.length];
}
```

Дополнительно `getRandomPositiveQuestion()` и `getRandomNegativeQuestion()` выбирают случайные вопросы из соответствующих наборов, а `getDayQuote(date)` детерминированно возвращает цитату дня.

## 9. Хранение данных

Все данные хранятся исключительно локально на устройстве: записи и настройки — в **AsyncStorage**, фотографии — в **файловой системе** устройства (на web — как base64).

**Особенности:**
- Данные не покидают устройство (нет backend).
- Сохраняются между сессиями приложения.
- При удалении приложения данные удаляются вместе с ним.
- Для резервного копирования есть экспорт/импорт записей в JSON-файл.

**Операции хранилища (storage.ts):**

| Функция | Описание |
|---|---|
| `formatDate(date)` | Форматирование даты в ключ "YYYY-MM-DD" |
| `saveDay(date, entry)` | Сохранить запись дня |
| `getDay(date)` | Получить запись по дате |
| `getAllDays()` | Все записи, от новых к старым |
| `getStreak()` | Вычислить текущую и лучшую серию |
| `getTags()` / `saveTags(tags)` | Получить/сохранить пользовательские теги |
| `addCustomTag(...)` / `deleteTag(...)` | Добавить/удалить тег |
| `getDiagnostics()` | Состояние локального хранилища |

**Операции с фото (photoStorage.ts):**

| Функция | Описание |
|---|---|
| `savePhoto(uri)` | Сохранить фото в файловую систему (web → base64), вернуть путь |
| `deletePhoto(path)` | Удалить файл фото |
