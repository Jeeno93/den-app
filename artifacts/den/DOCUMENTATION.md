## 1. Концепция продукта

**День** — это мобильное приложение для ежевечерней микро-рефлексии. Идея простая: каждый вечер пользователь отвечает на пять коротких вопросов о прошедшем дне и отмечает настроение одним из пяти уровней. Вся запись занимает 3–5 минут.

Приложение не требует регистрации, не отправляет данные на серверы и не показывает рекламу. Все записи хранятся локально на устройстве.

**Целевая аудитория:** люди, которые хотят развить привычку ежедневной рефлексии без сложных инструментов — без тегов, папок, форматирования и обязательств.

**Ключевой принцип:** минимализм. Один раз в день, пять вопросов, один клик на настроение — всё.

## 2. Сценарии использования

### Сценарий A — Ежедневная запись (основной)

**Контекст:** вечер, пользователь открывает приложение впервые за день.

1. Экран «Сегодня» показывает дату и день недели.
2. Пользователь видит пять эмодзи-кнопок настроения и выбирает одно.
3. Нажимает «Продолжить» — приложение переходит к вопросам.
4. Последовательно появляются пять карточек с вопросами:
   - Что сегодня узнал?
   - Кого встретил или вспомнил?
   - Что рассмешило?
   - Что раздражало?
   - Вопрос дня (меняется каждый день, всего 365 уникальных)
5. На каждой карточке пользователь вводит ответ и нажимает «Далее».
6. После последнего вопроса — анимированный экран подтверждения: «День записан. Увидимся завтра».
7. Опция «Посмотреть запись» позволяет сразу открыть сохранённый день.

### Сценарий B — Просмотр записи за сегодня

**Контекст:** пользователь уже заполнил дневник сегодня и возвращается в приложение.

1. Экран «Сегодня» автоматически переключается в режим просмотра.
2. Отображается бейдж «Записано» в правом верхнем углу.
3. Карточка настроения с эмодзи и цветовым акцентом.
4. Карточки с ответами на каждый вопрос (пустые ответы не показываются).

### Сценарий C — Просмотр прошлых дней через календарь

**Контекст:** пользователь хочет вспомнить, что происходило в конкретный день.

1. Открывает вкладку «Календарь».
2. Видит месячную сетку: дни с записями подсвечены цветными кружками (цвет = настроение).
3. Листает месяцы стрелками влево/вправо.
4. Нажимает на заполненный день — открывается детальная страница с полной записью.

### Сценарий D — Воспоминания и серии

**Контекст:** пользователь ведёт дневник уже несколько месяцев.

1. Открывает вкладку «Воспоминания».
2. Видит карточку с текущей серией дней (streak) и личным рекордом.
3. Блок «Год назад» показывает запись, сделанную ровно год назад (если есть).
4. Кнопка «Удиви меня» открывает случайный день из всей истории записей.

### Сценарий E — Настройка уведомлений

**Контекст:** пользователь хочет получать напоминание в удобное время.

1. Открывает вкладку «Настройки».
2. В разделе «Уведомления» нажимает на строку «Время напоминания».
3. Появляется барабанный выбор времени (часы и минуты с шагом 5 минут).
4. Нажимает «Сохранить» — уведомление перепланируется.

### Сценарий F — Смена темы

**Контекст:** пользователь хочет тёмное оформление ночью.

1. В «Настройках» в разделе «Оформление» выбирает «Тёмная», «Светлая» или «Системная».
2. Тема применяется мгновенно ко всем экранам и навигации.

## 3. Функциональность

### 3.1 Запись дня

- **Выбор настроения** — 5 уровней с эмодзи: 😞 Плохо, 😐 Нейтрально, 🙂 Хорошо, 😄 Отлично, 🤩 Супер.
- **5 фиксированных вопросов** — четыре постоянных и один «вопрос дня».
- **365 уникальных вопросов дня** — каждый день года имеет свой вопрос, назначенный по порядковому номеру дня в году.
- **Тактильный отклик** — лёгкая вибрация при выборе настроения, вибрация успеха при завершении записи (только iOS/Android).
- **Запись одна в день** — если запись уже сделана, форма заменяется режимом просмотра.
- **Анимация завершения** — появление с fade + scale при сохранении.

### 3.2 Просмотр записей

- **Режим просмотра сегодня** — автоматически активируется, если запись уже есть.
- **Детальная страница дня** — полная запись с карточками по каждому вопросу, цветом настроения и датой.
- **Скрытие пустых ответов** — если пользователь не ответил на вопрос, карточка не показывается.

### 3.3 Календарь

- **Месячная сетка** — классическая календарная раскладка с началом недели в понедельник.
- **Цветовые точки настроения** — каждый заполненный день окрашен в цвет настроения.
- **Выделение сегодня** — текущий день обведён рамкой акцентного цвета.
- **Пустые дни** — показаны серой точкой, некликабельны.
- **Навигация по месяцам** — стрелки назад/вперёд, обработка перехода год/год.
- **Легенда цветов** — внизу календаря объяснение шкалы настроений.

### 3.4 Воспоминания

- **Серия дней (streak)** — текущая непрерывная серия и лучшая серия за всё время.
- **Визуальная серия** — до 14 зелёных точек, символизирующих последовательные дни; если больше — показывается «+N».
- **Год назад** — запись ровно год назад от сегодняшней даты (если отсутствует — плейсхолдер с мотивационным сообщением).
- **Случайный день** — кнопка «Удиви меня» выбирает случайную запись из всей истории.

### 3.5 Настройки

- **Время уведомления** — настраиваемое время ежедневного напоминания (барабанный выбор с шагом 5 минут для минут).
- **Тема оформления** — три режима: Системная, Светлая, Тёмная. Активный режим помечен галочкой.
- **Экспорт данных** — заглушка, функция появится в следующей версии.
- **Версия приложения** — «День · v1.0.0» в подвале.

### 3.6 Уведомления

- Запрашиваются разрешения при первом запуске (iOS/Android).
- При получении разрешения уведомление планируется автоматически на 21:00 (по умолчанию).
- Каждый раз при открытии приложения (один раз в день) уведомление перепланируется с вопросом следующего дня.
- При смене времени пользователем — немедленная перепланировка.
- На web-версии уведомления не работают (платформенное ограничение).

## 4. Интерфейс

### 4.1 Навигация

Приложение построено на 4 вкладках (tab bar) в нижней части экрана:

| Вкладка | Иконка | Содержание |
|---|---|---|
| Сегодня | ☀️ (sun.min) | Форма записи или просмотр дня |
| Календарь | 📅 (calendar) | Месячный вид с точками настроений |
| Воспоминания | ✨ (sparkles) | Streak, год назад, случайный день |
| Настройки | ⚙️ (gearshape) | Тема, уведомления, экспорт |

На iOS используется BlurView для прозрачного эффекта tab bar. На Android и web — непрозрачный фон в цвет темы.

Отдельный роут `/day-detail` — детальная страница конкретного дня, открывается поверх tab bar (презентация `card`).

### 4.2 Экран «Сегодня» — состояния

Экран имеет 4 состояния, управляемых переменной `phase`:

**`mood`** — первичный экран (запись не сделана):
- Дата и день недели крупным шрифтом
- Заголовок «Как настроение? / Выбери одно из пяти»
- Ряд из 5 квадратных кнопок с эмодзи
- Кнопка «Продолжить» — неактивна до выбора настроения

**`questions`** — заполнение вопросов:
- Дата и день недели
- Карточка QuestionCard с вопросом, счётчиком (1/5), прогресс-точками, полем ввода и кнопкой «Далее»/«Готово»

**`done`** — анимация завершения:
- Центрированный зелёный круг с галочкой (fade + scale анимация)
- «День записан. / Увидимся завтра.»
- Кнопка «Посмотреть запись»

**`view`** — режим просмотра (запись уже есть):
- Хэдер с датой и бейджем «Записано»
- DayEntryView: карточка настроения + карточки ответов

### 4.3 Карточка вопроса (QuestionCard)

- Счётчик «1 / 5» и прогресс-точки (активная точка расширяется до 16px)
- Вопрос крупным шрифтом (20px, semibold)
- Многострочное текстовое поле (минимум 100px высоты)
- Кнопка «Далее» / «Готово» с иконкой

### 4.4 Карточка дня (DayEntryView)

- Карточка настроения: эмодзи + название уровня + «Настроение за день», фон — полупрозрачный цвет настроения
- Карточки ответов: мелкий заголовок вопроса + текст ответа; пустые ответы скрыты

### 4.5 Детальная страница (DayDetailScreen)

- Хэдер с кнопкой «назад» и датой в формате «2 апреля 2025»
- Полная запись через DayEntryView

- **Скруглённые углы** — радиус 16–20px для карточек, 14–16px для кнопок.

- **Elevation** — 2–4 для Android.
- **Акцент через прозрачность** — цвет настроения используется с прозрачностью `22` (hex) для фона карточек.
- **Анимации** — fade + spring при появлении (400ms timing + tension 60/friction 10), scale при завершении (tension 50/friction 8).
- **Строгий ритм** — горизонтальные отступы 20px, вертикальные gap 12–16px.

### 7.3 Поток данных

Все данные идут из AsyncStorage → компоненты читают через хуки и прямые вызовы storage.ts. Нет глобального стора — каждый экран загружает данные самостоятельно при фокусировке (`useFocusEffect`).

```
AsyncStorage
    ↑↓ storage.ts (saveDay / getDay / getAllDays / getStreak)
         ↑ useFocusEffect на каждом экране
              ↓ локальный useState
                   ↓ UI
```

### 7.4 Маршрутизация

Expo Router обрабатывает навигацию файловой системой:

- `/(tabs)/index` — главный экран (Сегодня)
- `/(tabs)/calendar` — Календарь
- `/(tabs)/memories` — Воспоминания
- `/(tabs)/settings` — Настройки
- `/day-detail?date=YYYY-MM-DD` — детальная страница (стек поверх таба)

Ключевое правило: только файлы внутри `app/` являются маршрутами. Все компоненты, контексты и утилиты вынесены в `src/`.

### 7.5 ThemeContext

```ts
type ThemeMode = "system" | "light" | "dark";

// Сохраняется в AsyncStorage["app_theme_mode"]
// Реактивно обновляет isDark во всём дереве
```

Компоненты используют `useTheme()` и берут нужную палитру: `const theme = isDark ? colors.dark : colors.light`.

### 7.6 NotificationContext

Логика планирования уведомлений:

1. При запуске → `requestPermissionsAsync()`
2. Если разрешено → читает `notification_time` из AsyncStorage
3. Если не планировалось сегодня (`notification_scheduled_date`) → планирует уведомление на завтра с вопросом следующего дня
4. При смене времени пользователем → сбрасывает дату планирования и перепланирует

## 8. Структура данных

### DayEntry — запись дня

```ts
interface DayEntry {
  date: string;          // "YYYY-MM-DD"
  mood: number;          // 1..5
  answers: DayAnswers;
  question: string;      // Вопрос дня на момент записи
  photo: string | null;  // Зарезервировано для будущей версии
}

interface DayAnswers {
  learned: string;       // Что узнал?
  met: string;           // Кого встретил?
  laughed: string;       // Что рассмешило?
  annoyed: string;       // Что раздражало?
  dayQuestion: string;   // Ответ на вопрос дня
}
```

### Ключи AsyncStorage

| Ключ | Содержание |
|---|---|
| `day_YYYY-MM-DD` | Сериализованный `DayEntry` |
| `app_theme_mode` | `"system"` / `"light"` / `"dark"` |
| `notification_time` | `"HH:MM"` |
| `notification_id` | ID запланированного уведомления |
| `notification_scheduled_date` | Дата последней перепланировки |

### Вопрос дня — getDayQuestion

```ts
function getDayQuestion(date: Date): string {
  const start = new Date(date.getFullYear(), 0, 0); // 31 дек прошлого года
  const dayOfYear = Math.floor((date - start) / 86400000);
  return questions[(dayOfYear - 1 + 365) % questions.length];
}
// Результат: questions[0] для 1 января, questions[364] для 31 декабря
```

## 9. Хранение данных

Все данные хранятся исключительно локально на устройстве в **AsyncStorage** — key-value хранилище React Native.

**Особенности:**
- Данные не покидают устройство
- Нет backend-зависимостей
- Данные сохраняются между сессиями приложения
- При удалении приложения данные удаляются вместе с ним
- Функция экспорта (резервное копирование) запланирована на следующую версию

**Операции хранилища:**

| Функция | Описание |
|---|---|
| `saveDay(date, entry)` | Сохранить запись дня |
| `getDay(date)` | Получить запись по дате |
| `getAllDays()` | Все записи, отсортированные от новых к старым |
| `getStreak()` | Вычислить текущую и лучшую серию |
