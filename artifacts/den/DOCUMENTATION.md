## 1. Концепция продукта

**День** — мобильное приложение для ежевечерней микро-рефлексии. Каждый вечер пользователь отмечает настроение, при желании отвечает на несколько коротких вопросов о прошедшем дне, добавляет теги места и активности, заметки, фото и поле «чем горжусь». Запись занимает от 30 секунд (режим «Быстро») до 5+ минут (режим «Глубоко»).

Приложение не требует регистрации, не отправляет содержимое записей на серверы и не показывает рекламу пользователю внутри продукта. Все записи и фотографии хранятся локально на устройстве (AsyncStorage + файловая система). На сервер уходят только обезличенные аналитические события (см. [раздел 10](#10-аналитика-и-атрибуция)).

**Целевая аудитория:** люди, которые хотят развить привычку ежедневной рефлексии без сложных инструментов — без папок, форматирования и обязательств.

**Ключевой принцип:** минимализм с глубиной по желанию. Три режима заполнения (Быстро / Стандарт / Глубоко) позволяют выбрать нужный уровень детализации, не отказываясь от привычки в загруженные дни.

## 2. Сценарии использования

### Сценарий A — Ежедневная запись (основной)

**Контекст:** вечер, пользователь открывает приложение впервые за день.

1. Экран «Сегодня» показывает дату и день недели.
2. Пользователь выбирает режим заполнения (Быстро / Стандарт / Глубоко) через `FillModeSwitcher` — по умолчанию используется последний выбранный режим.
3. Приложение последовательно проводит через фазы, зависящие от режима (см. [4.2](#42-фазы-заполнения-дня)):
   - **Настроение** — выбор одного из пяти уровней.
   - **Теги** — места и активности (во всех режимах).
   - **Вопросы** — пять карточек с вопросами (Стандарт и Глубоко).
   - **Заметки** — «Чем горжусь», свободное описание дня и до 3 фотографий (Стандарт и Глубоко).
   - **Задачи** — обзор вчерашних задач + планирование до 3 задач на завтра (все режимы).
   - **Глубоко** — уровень энергии (1–3) и трекер сна (время отбоя, подъёма, качество).
4. После завершения — анимированный экран подтверждения «День записан».
5. Опция «Посмотреть запись» открывает сохранённый день.

### Сценарий B — Просмотр и редактирование записи за сегодня

**Контекст:** пользователь уже заполнил дневник сегодня и возвращается в приложение.

1. Экран «Сегодня» автоматически переключается в режим просмотра.
2. Отображается карточка настроения с эмодзи и цветовым акцентом, полоска настроений за последние 7 дней (при ≥2 записях).
3. Чипы тегов места и активности.
4. Карточки с ответами на каждый вопрос (пустые ответы скрыты); под ответом — бейдж интенсивности, если был отмечен.
5. Карточка «Гордость дня», заметки «Итоги дня», фото дня, блоки энергии/сна и задач (если заполнялись в режиме «Глубоко»).
6. Любую карточку можно отредактировать прямо в режиме просмотра — изменения сохраняются сразу.
7. Кнопка «Поделиться днём» формирует картинку-карточку и открывает системное меню «Поделиться».
8. Стрелками можно перейти к предыдущему/следующему дню.

### Сценарий C — Просмотр прошлых дней через календарь

**Контекст:** пользователь хочет вспомнить, что происходило в конкретный день.

1. Открывает вкладку «Календарь».
2. Видит записи по месяцам, дни с записями подсвечены цветом настроения.
3. Нажимает на заполненный день — открывается детальная страница (`/day-detail`).
4. Нажимает на прошлый день без записи — открывается форма заполнения (`/day-fill`). Будущие дни некликабельны.
5. Через поиск ищет день по тексту ответов или заметок — результаты показываются списком с подсветкой совпадения.

### Сценарий D — Воспоминания и серии

**Контекст:** пользователь ведёт дневник уже несколько месяцев.

1. Открывает вкладку «Воспоминания».
2. Видит карточку текущей серии (streak) и личный рекорд.
3. Цитата дня (из `src/data/quotes.ts`, детерминированная по дню года).
4. Блоки «Неделю назад», «Месяц назад», «Год назад».
5. Ссылка на «Год в пикселях» — сетку из 365 квадратов, цвет = настроение дня; тап по ячейке открывает `/day-detail`.
6. Кнопка «Удиви меня» открывает случайный день из всей истории записей.

### Сценарий E — Письмо себе в будущее

**Контекст:** пользователь хочет оставить сообщение себе через N дней/месяцев.

1. Открывает «Письма себе» (`/letters`, ссылка из настроек).
2. Пишет текст письма и выбирает дату вскрытия — пресеты +1 неделя / +1 месяц / +3 месяца / +6 месяцев / +1 год, либо произвольная дата (минимум — через 7 дней).
3. Нажимает «Запечатать 💌» — письмо сохраняется закрытым (🔒), планируется push-уведомление на дату вскрытия.
4. До наступления даты письмо недоступно для чтения. Когда дата наступает, статус меняется на «Готово к прочтению» (✉️), появляется кнопка «Открыть письмо».
5. После открытия письмо помечается как прочитанное (📖) и доступно для повторного чтения в любой момент.

> Событие `letter_opened` физически не может произойти раньше чем через 7 дней после `letter_created` — это нужно учитывать при анализе конверсии в аналитике.

### Сценарий F — Настройка уведомлений

1. В «Настройках» включает ежедневное напоминание и задаёт время (по умолчанию 21:00, шаг 5 минут). Текст напоминания — вопрос дня на завтра.
2. Опционально включает уведомления «Воспоминания»: раз в 3 дня приложение находит случайную прошлую запись с настроением ≥3 (не старше недели, не моложе 3 дней) и присылает пуш с цитатой из неё.

### Сценарий G — Смена темы

Единственная доступная палитра — тёмная (см. [раздел 5](#5-дизайн-и-оформление)); переключатель темы в настройках зафиксирован на «Тёмная» до появления светлой версии.

### Сценарий H — Персонализация

1. **Теги** (`/tag-editor`) — добавление/удаление своих мест и активностей (эмодзи + название), изменение порядка.
2. **Вопросы** (`/question-editor`) — переопределение любого из 5 стандартных полей вопросов.

### Сценарий I — Экспорт и восстановление данных

1. В «Настройках» → «Данные» → «Экспортировать данные» — формируется JSON-файл, открывается меню «Поделиться».
2. «Восстановить данные» — выбор JSON-файла резервной копии и импорт записей.

### Сценарий J — Онбординг

При первом запуске показывается модальный онбординг (объясняет идею приложения). Факт прохождения сохраняется локально (`onboarding_done`), повторно не показывается.

## 3. Функциональность

### 3.1 Запись дня

- **Выбор настроения** — 5 уровней с эмодзи: 😞 Плохо, 😐 Нейтрально, 🙂 Хорошо, 😄 Отлично, 🤩 Супер.
- **Три режима заполнения** (переключаются `FillModeSwitcher`, последний выбор запоминается в `fill_mode`):

  | Режим | Фазы |
  |---|---|
  | **Быстро** | mood → tags → tasks |
  | **Стандарт** | mood → tags → questions → notes → tasks |
  | **Глубоко** | mood → tags → questions → notes → tasks → deep |

- **Вопросы дня** — пул из 392 вопросов, «вопрос дня» назначается по порядковому номеру дня в году (циклически). Дополнительно случайные «позитивный» и «негативный» вопросы. Любое из 5 стандартных полей можно переопределить через редактор вопросов.
- **Интенсивность переживаний** — для ответов (узнал, встретил, позитив, негатив, гордость) можно отметить силу переживания 1–3 с эмодзи и подписью.
- **Теги места и активности** — из настраиваемых списков (по умолчанию 10 мест, 12 активностей), доступны во всех режимах.
- **Задачи** — обзор вчерашних задач (отметка выполнено/нет) + планирование до 3 задач на завтра. Доступно во всех режимах, включая «Быстро».
- **Энергия и сон** (только режим «Глубоко») — уровень энергии 1–3, трекер сна (время отбоя, подъёма, качество).
- **Поле «Чем горжусь»** — опциональное достижение дня со своей интенсивностью.
- **Заметки дня** — свободное поле «Итоги дня».
- **Фото дня** — до 3 фотографий с камеры или из галереи.
- **Тактильный отклик** — вибрация при выборе настроения и при завершении (iOS/Android).
- **Запись одна в день** — если запись уже сделана, форма заменяется режимом просмотра.

### 3.2 Просмотр и редактирование записей

- Режим просмотра активируется автоматически, если запись за день уже есть; поддерживает навигацию к соседним дням.
- Inline-редактирование ответов, заметок, фото прямо в режиме просмотра.
- Бейджи интенсивности под ответами.
- Скрытие пустых ответов.
- «Поделиться днём» — генерация PNG-карточки (`react-native-view-shot`) и системное меню «Поделиться».
- Цитата дня внизу записи.

### 3.3 Календарь

- Список/сетка записей по месяцам, точки цвета настроения.
- Полнотекстовый поиск по ответам и заметкам с подсветкой совпадений.
- Заполнение прошлого дня открывает `/day-fill`; будущие дни некликабельны.
- Легенда цветов настроения.

### 3.4 Воспоминания

- Streak — текущая серия и лучшая серия за всё время.
- Цитата дня.
- Блоки «Неделю / месяц / год назад».
- «Год в пикселях» (`/year-pixels`).
- «Удиви меня» — случайный день из истории.

### 3.5 Письма себе (`/letters`)

- Time-capsule письмо: текст + дата вскрытия (7 дней – 1 год).
- Список писем со статусами: 🔒 заперто / ✉️ готово к прочтению / 📖 прочитано.
- Push-уведомление в день вскрытия.
- Удаление письма в любой момент (с отменой запланированного уведомления).

### 3.6 Настройки

- Ежедневное напоминание (время, шаг 5 минут).
- Уведомления «Воспоминания» (раз в 3 дня).
- Персонализация — редактор тегов, редактор вопросов.
- «Почему дневник работает» (`/why-diary`) — объяснение психологических механизмов.
- Данные — экспорт/импорт JSON, диагностика локального хранилища.
- Версия приложения в подвале.

### 3.7 Уведомления

- Запрашиваются разрешения (iOS/Android); на web не работают (платформенное ограничение).
- Ежедневное напоминание перепланируется мгновенно при смене времени.
- Уведомления «Воспоминания» — раз в 3 дня, на основе прошлых записей с настроением ≥3.
- Уведомление на дату вскрытия письма (`/letters`).

### 3.8 Онбординг

Модальный онбординг при первом запуске, повторно не показывается (`onboarding_done`).

## 4. Интерфейс

### 4.1 Навигация

4 вкладки (tab bar):

| Вкладка | Содержание |
|---|---|
| Сегодня | Форма записи (`DayFillFlow`) или просмотр дня |
| Календарь | Список записей по месяцам, поиск |
| Воспоминания | Streak, цитата, год в пикселях, недавние/прошлые записи, случайный день |
| Настройки | Уведомления, персонализация, письма, данные, диагностика |

Роуты вне табов (`card`-презентация поверх tab bar):

- `/day-detail?date=YYYY-MM-DD` — детальная страница дня.
- `/day-fill?date=YYYY-MM-DD` — форма заполнения конкретного дня (вызывается из календаря).
- `/year-pixels` — год в пикселях.
- `/letters` — письма себе.
- `/why-diary` — «Почему дневник работает».
- `/question-editor` — редактор вопросов.
- `/tag-editor` — редактор тегов мест и активностей.

### 4.2 Фазы заполнения дня

Общий компонент `DayFillFlow` используется и на вкладке «Сегодня», и на экране `/day-fill`. Набор фаз (`Phase = "mood" | "tags" | "questions" | "notes" | "tasks" | "deep" | "done"`) зависит от выбранного `FillMode`:

```ts
const PHASE_ORDER: Record<FillMode, Phase[]> = {
  quick:    ["mood", "tags", "tasks"],
  standard: ["mood", "tags", "questions", "notes", "tasks"],
  deep:     ["mood", "tags", "questions", "notes", "tasks", "deep"],
};
```

- **`mood`** — выбор настроения (5 эмодзи).
- **`tags`** — места и активности.
- **`questions`** — пять карточек `QuestionCard`, для части — выбор интенсивности (`IntensityPicker`).
- **`notes`** — гордость, заметки, до 3 фото (`NotesCard`).
- **`tasks`** — обзор вчерашних задач + планирование 3 задач на завтра.
- **`deep`** — энергия (1–3) и данные о сне (`DeepBlocks`).
- **`done`** — анимация завершения «День записан».

После завершения экран «Сегодня» переключается в фазу `view` — просмотр/редактирование с навигацией по дням.

### 4.3 Ключевые компоненты интерфейса

- **QuestionCard** — счётчик, прогресс-точки, вопрос, многострочное поле, кнопки «Назад»/«Далее».
- **DayEntryView (DayEntry.tsx)** — карточка настроения, чипы тегов, карточки ответов с inline-редактированием и бейджами интенсивности, «Гордость дня», заметки, фото, задачи, энергия/сон, кнопка «Поделиться днём», цитата дня.
- **DayDetailScreen** — хэдер с датой + `DayEntryView`.
- **ShareCard** — компактная визуальная карточка дня, захватывается в PNG для шеринга.
- **FillModeSwitcher** — переключатель режима (свёрнутый/развёрнутый).
- **DeepBlocks** — блоки энергии и сна для режима «Глубоко».
- **DeepNudgeBanner** — баннер-подсказка «попробуй режим Глубоко» (можно скрыть, `deep_nudge_dismissed`).

## 5. Дизайн и оформление

### 5.1 Цветовая палитра

Единая тёмная палитра (`constants/colors.ts`; `light` и `dark` в коде указывают на один и тот же объект — отдельной светлой темы сейчас нет):

| Токен | HEX | Назначение |
|---|---|---|
| background | `#06080B` | Фон экранов |
| card | `#0D1117` | Фон карточек |
| secondary / muted | `#10141A` | Вторичные поверхности, поля ввода |
| primary / accent / tint | `#5EE6A8` | Акцент (зелёный) |
| foreground / text | `#FFFFFF` | Основной текст |
| mutedForeground | `#6B7585` | Вспомогательный текст, подписи |
| border / input | `rgba(255,255,255,0.08)` | Рамки |
| destructive | `#EF4444` | Деструктивные действия |

### 5.2 Цвета настроений

| Уровень | Эмодзи | Название | Цвет |
|---|---|---|---|
| 1 | 😞 | Плохо | `#7B8FA1` |
| 2 | 😐 | Нейтрально | `#A8B5C1` |
| 3 | 🙂 | Хорошо | `#90C8A8` |
| 4 | 😄 | Отлично | `#5BAD8F` |
| 5 | 🤩 | Супер | `#5EE6A8` |

### 5.3 Ключевые элементы

- **CTA-кнопки** — `LinearGradient` `#214E3C → #4DD894 → #2F7D58`, `borderRadius: 28`.
- **MoodPicker** — кружки 52×52, градиент белый→`#C8D1D9`, активный: рамка `#5EE6A8` + свечение + `scale: 1.15`.
- **Карточки** — `backgroundColor: #0D1117`, `borderWidth: 1`, `borderColor: rgba(255,255,255,0.08)`, `elevation: 20`, радиус 16–20px.
- **Типографика** — шрифт **Inter** (Google Fonts): Regular 400, Medium 500, SemiBold 600, Bold 700.
- **Анимации** — fade + scale при появлении и завершении.
- **Ритм** — горизонтальные отступы 20px, вертикальные gap 12–20px.

## 8. Структура данных

### DayEntry — запись дня (`src/storage/storage.ts`)

```ts
type Intensity = 1 | 2 | 3 | null;
type FillMode = "quick" | "standard" | "deep";

interface QuestionAnswer {
  question: string;
  answer: string;
  category: string;
}

interface DayAnswers {
  learned: string;             // Что узнал?
  met: string;                 // Кого встретил/вспомнил?
  positive: QuestionAnswer;    // Случайный позитивный вопрос + ответ
  negative: QuestionAnswer;    // Случайный негативный вопрос + ответ
  dayQuestion: string;         // Ответ на вопрос дня
}

interface SleepData {
  bedtime: string;              // "23:30" или ""
  wakeTime: string;             // "07:00" или ""
  quality: number | null;       // 1-3
}

interface TaskItem {
  id: string;
  text: string;
  done: boolean;
}

interface HabitItem {           // будущая фича, поле сохраняется для совместимости
  id: string;
  label: string;
  done: boolean;
}

interface DayEntry {
  date: string;                 // "YYYY-MM-DD"
  mood: number;                 // 1..5
  answers: DayAnswers;
  question: string;             // Вопрос дня на момент записи
  notes: string;                // Свободные «Итоги дня»
  photos: string[];             // До 3 путей/URI к фото
  proud: string;                // Чем горжусь
  learned_intensity: Intensity;
  met_intensity: Intensity;
  positive_intensity: Intensity;
  negative_intensity: Intensity;
  proud_intensity: Intensity;
  places: string[];             // ID выбранных тегов мест
  activities: string[];         // ID выбранных тегов активностей
  // Опционально: режимы «Быстро/Стандарт/Глубоко»
  fillMode?: FillMode;
  energy?: number | null;       // уровень энергии 1-3 (Глубоко)
  sleep?: SleepData | null;     // трекер сна (Глубоко)
  habits?: HabitItem[];         // трекер привычек — будущая фича
  tasksForTomorrow?: TaskItem[];// до 3 задач на завтра
  tasksReviewed?: TaskItem[];   // вчерашние задачи, отмеченные сегодня
}
```

**Обратная совместимость:** при чтении старых записей `parseEntry` нормализует данные — одиночное `photo` превращается в массив `photos`, устаревшие ключи `laughed`/`annoyed` мапятся в `positive`/`negative`, отсутствующие поля получают значения по умолчанию.

## 9. Хранение данных

Все содержательные данные хранятся исключительно локально: записи и настройки — в **AsyncStorage**, фотографии — в **файловой системе** устройства (на web — base64). Бэкенда для пользовательских данных нет.

**Особенности:**
- Данные не покидают устройство.
- Сохраняются между сессиями приложения.
- Удаляются вместе с приложением.
- Для резервного копирования — экспорт/импорт JSON.

**Операции хранилища (storage.ts):**

| Функция | Описание |
|---|---|
| `formatDate(date)` | Форматирование даты в ключ "YYYY-MM-DD" |
| `saveDay(date, entry)` | Сохранить запись дня |
| `getDay(date)` | Получить запись по дате |
| `getAllDays()` | Все записи, от новых к старым |
| `getStreak()` | Вычислить текущую и лучшую серию |
| `getTags()` / `saveTags(tags)` | Получить/сохранить пользовательские теги |
| `addCustomTag(...)` / `deleteTag(...)` | Добавить/удалить тег |
| `getFillMode()` / `saveFillMode(mode)` | Получить/сохранить последний режим заполнения |
| `getCustomQuestions()` / `hasAnyCustomQuestion()` | Пользовательские переопределения вопросов |
| `getLetters()` / `upsertLetter(letter)` / `deleteLetter(id)` | CRUD писем себе в будущее |
| `getDiagnostics()` | Состояние локального хранилища |

**Операции с фото (photoStorage.ts):**

| Функция | Описание |
|---|---|
| `savePhoto(uri)` | Сохранить фото в файловую систему (web → base64), вернуть путь |
| `deletePhoto(path)` | Удалить файл фото |

> ⚠️ Известное ограничение: на Android есть исторический потолок ~6 МБ (`SQLITE_FULL`) из-за старых записей с inline base64-фото. Новые большие блобы должны идти только через файловую систему, не через AsyncStorage напрямую (см. `.agents/memory/den-storage-sqlite-full.md`).

## 10. Аналитика и атрибуция

Продукт использует два независимых аналитических SDK с разными задачами:

### 10.1 Amplitude (`@amplitude/analytics-react-native`)

Основная продуктовая аналитика — трекает поведенческие события (`app_opened`, `fill_started/completed/abandoned`, `letter_created/opened`, `calendar_viewed`, `memories_viewed`, `streak_milestone/broken`, `onboarding_completed`, `notification_tapped`, `year_pixels_viewed`, `settings_viewed`, `deeplink_params_received/failed` и др.). Работает на всех платформах. Проект в Amplitude — **Den** (appId `829084`).

### 10.2 AppMetrica (`@appmetrica/react-native-analytics`) — только Android

Файлы: `src/analytics/appMetrica.ts` (no-op фолбэк для web/iOS) и `src/analytics/appMetrica.android.ts` (реальная реализация, подключается автоматически Metro-бандлером на Android).

**Назначение — атрибуция установок и оптимизация рекламных кампаний в Яндекс.Директ:**

1. `initAppMetrica()` активирует AppMetrica по ключу из `EXPO_PUBLIC_APPMETRICA_API_KEY` и на первом запуске после установки запрашивает Google Play Install Referrer (`requestDeferredDeeplinkParameters`) — это единственный момент, когда эти данные доступны (`NOT_A_FIRST_LAUNCH` на всех последующих запусках).
2. Полученные параметры атрибуции пробрасываются в Amplitude как user properties `acq_<key>` через `Identify.setOnce`, событие `deeplink_params_received`/`deeplink_params_failed` фиксирует результат.
3. `reportGoalEvent(name)` отправляет в AppMetrica *отдельное* от Amplitude событие-цель. Единственный вызов сейчас — `reportGoalEvent("fill_completed")` в `DayFillFlow.tsx` при завершении записи дня. Комментарий в коде прямо объясняет цель: **дать рекламной платформе (Яндекс.Директ) сигнал реального вовлечения, а не только факт установки**, чтобы кампании могли оптимизироваться на «дошёл до заполнения дня», а не только на инсталлы.

**Известная проблема (зафиксирована в коде комментарием разработчика):** `acq_*` свойства атрибуции не долетали до Amplitude в течение ~5 дней реальных установок, при этом `onFailure`-ветка логирует ошибку только локально — то есть в текущем виде **нет видимости в самой Amplitude**, сработала ли атрибуция Яндекс.Директ для конкретного пользователя. Событие `deeplink_params_failed` есть в данных Amplitude (12 за последние 10 дней в связке с `notification_tapped` — не путать), но без разбивки по коду ошибки в самой аналитике сложно понять, что именно ломается.
